# foma/topsort.c

> [spec:foma:def:topsort.fsm-topsort-fn]
> struct fsm *fsm_topsort (struct fsm *net)

> [spec:foma:sem:topsort.fsm-topsort-fn]
> Topologically sorts the states of `net` (renumbering states in topological order and rewriting the state array) while simultaneously counting accepting paths, using Kahn's algorithm over inverse-arc counts. If the graph is cyclic, the state array is left untouched and only pathcount/is_loop_free flags are set. NULL input → NULL. Returns `net` (modified in place) in all other cases.
> Setup: call fsm_count(net) to refresh counts. Allocate per-state arrays sized net->statecount: statemap (int, init -1; first line index of each state in the fsm_state array), order (int, init 0; treatment order → old state number), pathcount (long long, init 0), newnum (int, uninit; old state number → new number), invcount (unsigned short, init 0; number of incoming arcs — silently wraps past 65535), treated (unsigned char, init 0).
> Pass 1 over the line array (terminated by a line with state_no==-1): count lines lc; for each line with target != -1, increment invcount[target], and if state_no==target (self-loop) immediately mark cyclic: net->pathcount=PATHCOUNT_CYCLIC (-1), net->is_loop_free=0, and jump to cleanup (free the six arrays, clear the global int stack, return net). Record statemap[state_no]=line index on the first line seen for each state.
> Main loop: treatcount=net->statecount; clear the global int stack and push state 0 (the initial state is assumed to be 0 with no untreated predecessors); grand_pathcount=0; pathcount[0]=1; overflow=0. With counter i starting at 0, while the stack is non-empty: pop curr_state; treated[curr_state]=1; order[i]=curr_state; newnum[curr_state]=i; treatcount--; then walk that state's consecutive lines starting at statemap[curr_state] while line->state_no==curr_state. For each line with target != -1: decrement invcount[target]; if !overflow, pathcount[target] += pathcount[curr_state] and set overflow=1 if the sum went negative (signed 64-bit wrap); if treated[target]==1 (arc into an already-treated state — includes back-edges) mark cyclic as above and jump to cleanup; if invcount[target] reached 0, push target. Increment i per popped state.
> After the loop, if treatcount > 0 (some states never reached invcount 0 — a cycle not caught above) mark cyclic as above and jump to cleanup.
> Acyclic rebuild: malloc a new line array of lc+1 fsm_state entries. For i over 0..statecount-1 in treatment order: curr_state=order[i]; locate its lines via statemap. If the state's first line has final_state==1 and !overflow, grand_pathcount += pathcount[curr_state], setting overflow=1 if it goes negative. Copy each of the state's lines to the next output slot j via add_fsm_arc(new, j, newnum[state_no], in, out, target==-1 ? -1 : newnum[target], final_state, start_state). Append the terminator line (-1,-1,-1,-1,-1,-1). Set net->states=new array, free the old array; net->pathcount=grand_pathcount, net->is_loop_free=1; if overflow, net->pathcount=PATHCOUNT_OVERFLOW (-2) instead.
> Cleanup (all exits): free statemap/order/pathcount/newnum/invcount/treated, clear the global int stack, return net.
> Resulting properties: state numbers equal topological rank (initial state becomes 0; every arc goes from a lower to a higher number); lines remain grouped per state, states emitted in the new numeric order; net->pathcount is the number of distinct accepting paths from the initial state, PATHCOUNT_CYCLIC for cyclic nets, PATHCOUNT_OVERFLOW if the count exceeded signed 64-bit range. Assumes state 0 exists and is the start; states unreachable from 0 whose invcount never drops to 0 cause the net to be reported cyclic even if it is not.
